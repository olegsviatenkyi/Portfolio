Airtable, Zapier and Sender Automation System

👤 Role: Airtable & Zapier Automation Specialist

🔍 Project Overview

This project focused on improving an existing event management system built with Airtable, Zapier, JotForm and Sender. The system handled more than 180 participants across several events, but many processes were difficult to manage and depended on manual checks.

The main goal was to make medical form tracking more accurate, reduce manual work, and give the event team a clear view of submitted, missing and flagged forms.

🎯 Business Challenge

The existing setup was working, but the logic was difficult to maintain and participant matching was not always reliable.

  • Several participants could share the same email address
  • Old form submissions did not contain a unique ticket code
  • Duplicate records existed in Airtable and JotForm
  • Standard and risky medical responses had to be separated
  • The team needed clear views for missing and flagged forms
  • Sender and Airtable synchronization issues were difficult to diagnose

🛠️ Solution

I improved the existing workflow by introducing ticket code matching and building a dedicated JotForm to Airtable automation in Zapier.

The solution included:

  • Adding a ticket code field to the medical JotForm
  • Passing the unique ticket code through personalized form links
  • Updating the Sender email template to keep the ticket code in the URL
  • Building a Zap that finds the correct Airtable record by ticket code
  • Separating submissions into Submitted and Flagged paths
  • Updating medical status and submission date automatically
  • Creating dedicated Airtable views for tracking and review
  • Processing historical submissions and documenting exceptions

⚙️ Automation Structure

JotForm Submission
The workflow starts when a participant submits the medical questionnaire.

Participant Matching
Zapier searches Airtable using the unique ticket code instead of email. This improves accuracy when several participants use the same email address.

Response Routing
The Zap checks the medical answers and sends each submission through one of two controlled paths:

  • Submitted: no response requires manual review
  • Flagged: one or more answers require attention

Airtable Update
The correct participant record is updated with the medical form status and submission date.

🔑 Ticket Code Matching

Email matching was replaced with ticket code matching for the medical form workflow. Each participant receives a personalized form link containing their unique ticket code.

This made the process more reliable and reduced the risk of updating the wrong participant record.

📋 Airtable Tracking Views

Dedicated Airtable views were created to make daily work easier for the event team.

  • Medical Form Tracking
  • Medical Forms Not Started
  • Flagged Medical Submissions

These views allow the team to focus on exceptions instead of checking every participant manually.

🗂 Historical Data Cleanup

Older JotForm submissions did not include ticket codes, so they had to be reviewed and matched with Airtable records.

  • Existing submissions were matched with participant records
  • Medical status and submission dates were updated
  • Duplicate and missing records were documented
  • Unclear cases were separated for manual review

🔎 System Diagnostics

I also reviewed several existing automations and synchronization issues in the old system.

  • Verified the Adult Waiver Reminder logic
  • Located a Sender synchronization failure inside the Zapier action
  • Confirmed how changed email addresses were handled in Sender
  • Investigated missing team and insurance links
  • Identified weak skipper and crew matching logic

✅ Result

  • More accurate participant matching
  • Automatic medical form status updates
  • Clear separation of standard and flagged submissions
  • Less manual checking before each event
  • Better visibility of missing forms and problem cases
  • A clearer list of weak points for future system improvements

📦 Final Setup

  • Published JotForm to Airtable Zap
  • Submitted and Flagged routing logic
