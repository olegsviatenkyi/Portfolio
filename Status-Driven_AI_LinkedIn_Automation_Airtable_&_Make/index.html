Status-Driven AI LinkedIn Automation (Airtable & Make)

üë§ Role: Automation & AI Systems Builder (Make.com, Airtable)

üîç Project Overview

This project is a status-driven AI content automation system designed to generate, review, and publish LinkedIn posts in a controlled and repeatable way.

The system combines Airtable as a content database, Make.com for orchestration, OpenAI for text generation, and Replicate for image generation, with mandatory human review stages before publishing.

The main goal was to eliminate chaotic manual posting while preserving quality control and editorial oversight.

‚ùå Problem

Manual LinkedIn content creation and publishing has several common issues:

  • Content creation is time-consuming and inconsistent
  • Posts are generated but forgotten or published late
  • No clear approval or moderation stages
  • Difficult to scale posting without losing quality
  • Images and copy are often disconnected or rushed

Teams and solo creators need automation, but not blind auto-posting.

‚úÖ Solution

I built a status-based automation pipeline where each step is explicitly controlled by Airtable statuses.

AI assists with content creation, but humans stay in control at key decision points.

The system ensures:

  • Clear content lifecycle
  • No accidental publishing
  • Visual consistency between text and image
  • Scalable posting without daily manual effort

üß† System Architecture

  1. Topic Intake & Text Generation
    • Topics are submitted via an Airtable Form
    • Initial status: In progress
    • A Make.com scenario:
      • Reads the topic from Airtable
      • Sends it to an OpenAI Assistant
      • Receives generated LinkedIn post text
      • Saves the text back to Airtable
      • Updates status to Waiting for tools
  2. Tool Selection & Image Generation
    • Two primary tools aren selected manually in Airtable
    • Status is set to Ready for image
    • The same Make.com scenario:
      • Reads topic + selected tools
      • Sends structured input to Replicate
      • Generates a LinkedIn-optimized image
      • Saves the image back to Airtable
      • Updates status to Moderation
  3. Human Review & Approval
    • Post text and image are reviewed directly in Airtable
    • If everything is satisfactory, status is set to Ready to Publish

    This guarantees zero blind publishing.

  4. Automated Publishing to LinkedIn
    • A separate Make.com scenario is triggered
    • The scenario:
      • Pulls the approved post text and image
      • Publishes the content to LinkedIn automatically

üìä Result

  • Fully controlled AI-assisted content pipeline
  • Clear editorial workflow with approval gates
  • Consistent posting without daily manual work
  • No duplicate posts or accidental publishing
  • Easy to scale to higher posting frequency

The system balances automation speed with human quality control.
