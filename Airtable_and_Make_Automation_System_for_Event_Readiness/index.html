Airtable and Make Automation System for Event Readiness

👤 Role: Airtable & Make Automation Specialist

🔍 Project Overview

This project focused on building a new event readiness system using Airtable, Make, JotForm and Postmark. The system was created to manage participant records, teams, compliance forms, onboarding emails and event readiness in one central workflow.

The main goal was to reduce manual tracking and create a reliable system that could support multiple rally events at the same time.

🎯 Business Challenge

The event team needed a clear and reliable way to manage participants, teams and required readiness steps before each rally.

  • Participant and team data had to be managed in one central place
  • Skippers, crew and under 18 participants needed clear matching logic
  • Compliance forms had to update the correct Airtable records
  • Different forms were required for adults, under 18 participants and skippers
  • Email reminders had to be sent at the right time for each participant type
  • The team needed clear views for missing, submitted and flagged records
  • The system had to support several rally events without mixing records

🛠️ Solution

I built a new Airtable-based readiness system and connected it with Make automations for bookings, forms, emails and compliance tracking.

The solution included:

  • Creating a structured Airtable base for events, participants, teams and compliance
  • Building Make automations to process new ticket purchases
  • Creating participant records from Ticket Tailor bookings
  • Automatically linking skippers, crew and under 18 participants to teams
  • Creating compliance records for each participant
  • Connecting JotForm submissions to Airtable compliance status fields
  • Sending onboarding and readiness emails through Postmark
  • Adding reminder logic for missing forms and incomplete steps
  • Creating Airtable views for daily review and issue tracking

⚙️ Automation Structure

Ticket Purchase
The workflow starts when a participant buys a ticket. Make receives the booking data and checks the ticket status, event code and participant role.

Participant Creation
The system creates or updates the participant record in Airtable and assigns the correct role, event, ticket code and related details.

Team Matching
Skippers are used to create teams, while crew and under 18 participants are linked to the correct skipper or guardian where possible.

Compliance Tracking
A related compliance record is created for each participant. This record tracks waivers, medical forms, team details, insurance and induction status.

Email Automation
Postmark emails are sent based on participant status and event timing. Each email contains the correct form link with the participant ticket code.

🔑 Ticket Code Matching

Ticket code matching was used as the main way to connect form submissions with the correct Airtable records.

This made the process more reliable than email matching, especially when multiple participants used the same email address or when under 18 participants were managed by a guardian.

📋 Airtable Tracking Views

Dedicated Airtable views were created to help the event team review the system quickly and focus on records that need attention.

  • Participants by event
  • Skippers, crew and under 18 participants
  • Teams by rally
  • Incomplete compliance records
  • Flagged compliance records
  • Missing waiver, medical, insurance and induction records
  • Communication log by event

These views make it easier to manage multiple events without mixing participant, team and compliance data.

📨 Email & Reminder Flow

The system uses Postmark templates for onboarding and readiness emails. Emails are triggered based on ticket purchase, participant role and event timeline.

  • Welcome and resources emails after ticket purchase
  • Waiver form emails for adults and under 18 participants
  • Medical questionnaire emails
  • Team and boat details emails for skippers
  • Insurance and licence upload emails
  • Induction emails before the rally
  • Reminder emails for incomplete steps

🔎 Testing & Launch Review

Before launch, I tested the main workflows to make sure the system correctly processed bookings, created participant records, updated compliance statuses and sent the right emails.

  • Tested booking automation from Ticket Tailor to Airtable
  • Verified participant creation for skippers, crew and under 18 participants
  • Checked team matching and participant linking logic
  • Tested JotForm submissions and compliance status updates
  • Verified Postmark email templates and dynamic form links
  • Checked reminder scenarios for missing readiness steps
  • Reviewed Airtable views for participant, team and compliance tracking

✅ Result

  • Clear tracking of waiver, medical, team details, insurance and induction status
  • Automated onboarding and reminder emails through Postmark
  • Better visibility of missing and flagged records
  • Clear structure for managing multiple event launches
