HubSpot CRM Automation System with Google Sheets Integration

👤 Role: HubSpot CRM & Make.com Automation Specialist

🔍 Project Overview

This project focused on building a structured automation system between Google Sheets and HubSpot CRM using Make.com. The goal was to automate the creation, updating, and synchronization of Contacts, Companies, and Deals while reducing manual CRM work and improving data consistency.

The solution included upsert logic, validation rules, association workflows, centralized logging, and a separate update scenario for changes coming from Google Sheets.

🎯 Business Challenge

The business needed a more reliable way to manage CRM data stored in Google Sheets and keep HubSpot records accurate and structured.

  • Manual creation and updating of Contacts, Companies, and Deals
  • Risk of duplicate or incomplete records
  • Missing associations between Deals, Contacts, and Companies
  • No clear visibility into automation results and failures
  • Need to handle invalid or missing data without breaking scenarios
  • Need for centralized logging for support and troubleshooting

The system had to support a scalable workflow where spreadsheet-driven operations could update HubSpot consistently and safely.

🛠️ Solution

I designed and implemented a Make.com automation system connecting Google Sheets with HubSpot CRM through structured scenario logic.

The solution includes:

  • Contacts upsert flow with search by email
  • Companies upsert flow with search by domain
  • Deals upsert flow with search by external ID
  • Router-based branching for create vs update logic
  • Automatic deal associations to Contact and Company
  • Validation checks before record processing
  • Centralized logging of success and error events into Google Sheets
  • Separate scenario for form submission logging
  • Separate scenario for handling updates triggered by Sheet changes

🗂 Automation Structure

Contacts Upsert
A dedicated scenario processes new rows from Google Sheets and searches HubSpot Contacts by email. If the contact already exists, the record is updated. If no match is found, a new contact is created.

Companies Upsert
A separate scenario handles Companies using domain as the main lookup key. This flow updates existing companies when a match is found and creates new ones when needed.

Deals Upsert
Deals are processed with external ID as the primary reference. This allows the system to identify existing deals, update them correctly, or create new ones without relying on unstable manual matching.

🔗 Association Logic

A dedicated part of the automation handles relationships between CRM objects.

  • Automatic association of Deals with Contacts
  • Automatic association of Deals with Companies

This helps preserve CRM structure and ensures deals remain connected to the correct people and businesses.

⚙️ Routing & Fallback Logic

Routers were used to split scenario execution into controlled branches depending on record state and validation results.

  • Update route when an existing record is found
  • Create route when no match exists
  • Fallback handling for missing Contact
  • Fallback handling for missing Company
  • Default deal stage handling when needed

This approach improves reliability and keeps scenarios readable and maintainable.

✅ Data Validation & Error Handling

Validation was added before key actions to prevent incorrect data from entering the CRM.

  • Email validation
  • Domain validation
  • Required field validation for Deals

Error handling routes were also implemented so the scenarios do not stop when a record fails. Instead of breaking the full flow, issues are redirected into the logging layer for review.

🔁 Retry Stability

Retry-oriented logic was implemented to improve resilience in case of temporary HubSpot API issues. This helped reduce scenario interruptions and made the automation more stable during repeated operations.

📋 Logging System

A centralized logging system was built in Google Sheets to track automation activity.

  • Timestamp
  • Scenario name
  • Record key
  • Action
  • Status
  • Error message

This makes it easier to monitor performance, review failures, and troubleshoot record-level issues.

🔄 Sheets-to-HubSpot Update Logic

In addition to new row processing, a separate scenario was built to handle updates triggered by changes inside Google Sheets. This allows operational edits made in Sheets to be pushed back into HubSpot through a controlled update path.

📝 HubSpot Form Submission Logging

A dedicated scenario was added to capture HubSpot form submissions and log them into Google Sheets. This provided an additional tracking layer for incoming submissions and improved operational visibility.

📦 Final Setup

  • Automated Contacts sync from Google Sheets to HubSpot
  • Automated Companies sync from Google Sheets to HubSpot
  • Automated Deals sync from Google Sheets to HubSpot
  • Deal association logic for Contact and Company linking
  • Validation and fallback handling
  • Centralized Google Sheets logging system
  • Separate HubSpot form submission logging flow
  • Separate Sheets changes to HubSpot update scenario
  • Structured scenario naming for maintainability
